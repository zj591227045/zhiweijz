# 只为记账 - UI设计风格

## 1. 设计原则

"只为记账"应用的UI设计遵循以下核心原则:

1. **简洁直观**: 界面简洁明了，减少视觉干扰，让用户专注于记账和财务管理
2. **移动优先**: 优先考虑移动设备的用户体验，确保在小屏幕上操作流畅
3. **一致性**: 保持视觉元素和交互模式的一致性，降低学习成本
4. **即时反馈**: 为用户操作提供清晰的反馈，增强用户信心
5. **无障碍**: 确保应用可被各种用户使用，包括视力障碍用户

## 2. 主题系统

"只为记账"应用采用灵活的主题系统，支持多套内置主题和自定义主题功能。

### 2.1 主题架构

主题系统基于CSS变量和Tailwind CSS构建:

- **基础变量**: 所有主题共享的基础CSS变量
- **主题变体**: 基于基础变量的不同主题实现
- **主题切换**: 通过切换CSS变量实现无刷新主题切换
- **主题存储**: 用户主题偏好保存在本地存储中
- **主题继承**: 支持主题间的继承和覆盖

### 2.2 内置主题

应用提供以下内置主题:

1. **默认主题 (Default)**: 蓝色基调的明亮主题
2. **暗色主题 (Dark)**: 深色背景的暗色主题
3. **蓝色主题 (Blue)**: 深蓝色调的主题
4. **绿色主题 (Green)**: 自然绿色调的主题
5. **紫色主题 (Purple)**: 优雅紫色调的主题

### 2.3 默认主题配色方案

以下是默认主题的配色方案:

#### 2.3.1 主色调

- **主色 (Primary)**: `#3B82F6` - 蓝色，用于主要按钮、强调元素和品牌标识
- **辅助色 (Secondary)**: `#10B981` - 绿色，用于辅助元素和成功状态

#### 2.3.2 功能色

- **成功 (Success)**: `#22C55E` - 绿色，表示操作成功、收入等积极状态
- **警告 (Warning)**: `#F59E0B` - 橙色，表示警告、预算接近上限等状态
- **错误 (Error)**: `#EF4444` - 红色，表示错误、支出等负面状态
- **信息 (Info)**: `#3B82F6` - 蓝色，表示提示信息

#### 2.3.3 中性色

- **背景 (Background)**: `#F9FAFB` - 浅灰色，页面背景
- **卡片背景 (Card)**: `#FFFFFF` - 白色，卡片和面板背景
- **文本主色 (Text Primary)**: `#1F2937` - 深灰色，主要文本
- **文本次色 (Text Secondary)**: `#6B7280` - 中灰色，次要文本
- **边框 (Border)**: `#E5E7EB` - 浅灰色，分隔线和边框

#### 2.3.4 渐变色

- **主渐变**: `linear-gradient(135deg, #3B82F6 0%, #2563EB 100%)` - 用于特殊强调元素
- **成功渐变**: `linear-gradient(135deg, #10B981 0%, #059669 100%)` - 用于收入相关元素
- **警告渐变**: `linear-gradient(135deg, #F59E0B 0%, #D97706 100%)` - 用于警告相关元素

### 2.4 主题变量结构

所有主题共享相同的变量结构，确保一致性和可互换性:

:root {

### 2.5 主题自定义

用户可以自定义主题并保存:

1. **颜色选择器**: 用户可以调整主要颜色
2. **预览功能**: 实时预览主题更改效果
3. **保存功能**: 将自定义主题保存到本地存储
4. **导出/导入**: 支持主题配置的导出和导入
5. **重置功能**: 恢复到内置主题设置

## 3. 字体系统

### 3.1 字体家族

- **主字体**: `-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif` - 系统默认字体，确保在各平台上的最佳显示效果

### 3.2 字体大小

- **超大标题 (2xl)**: `24px` - 页面主标题
- **大标题 (xl)**: `20px` - 区块标题
- **中标题 (lg)**: `18px` - 卡片标题
- **小标题 (md)**: `16px` - 次要标题
- **正文 (base)**: `14px` - 主要内容
- **小字 (sm)**: `12px` - 辅助信息
- **超小 (xs)**: `10px` - 标签和徽章

### 3.3 字重

- **常规 (Regular)**: `400` - 正文文本
- **中等 (Medium)**: `500` - 次要标题
- **半粗 (Semibold)**: `600` - 主要标题和强调文本
- **粗体 (Bold)**: `700` - 特别强调

### 3.4 行高

- **紧凑 (Tight)**: `1.2` - 标题
- **正常 (Normal)**: `1.5` - 正文
- **宽松 (Loose)**: `1.8` - 长文本段落

## 4. 设计系统

### 4.1 间距系统

使用4的倍数作为基础间距单位:

- **xs**: `4px` - 最小间距
- **sm**: `8px` - 小间距
- **md**: `16px` - 中等间距
- **lg**: `24px` - 大间距
- **xl**: `32px` - 特大间距
- **2xl**: `48px` - 超大间距

### 4.2 圆角

- **none**: `0px` - 无圆角
- **sm**: `4px` - 小圆角
- **md**: `8px` - 中等圆角
- **lg**: `12px` - 大圆角
- **xl**: `16px` - 特大圆角
- **full**: `9999px` - 完全圆形

### 4.3 阴影

- **sm**: `0 1px 2px rgba(0, 0, 0, 0.05)` - 轻微阴影
- **md**: `0 4px 6px -1px rgba(0, 0, 0, 0.1), 0 2px 4px -1px rgba(0, 0, 0, 0.06)` - 中等阴影
- **lg**: `0 10px 15px -3px rgba(0, 0, 0, 0.1), 0 4px 6px -2px rgba(0, 0, 0, 0.05)` - 大阴影
- **xl**: `0 20px 25px -5px rgba(0, 0, 0, 0.1), 0 10px 10px -5px rgba(0, 0, 0, 0.04)` - 特大阴影

### 4.4 基础组件规范

#### 按钮

- **主要按钮**: 蓝色背景，白色文本，`12px`圆角
- **次要按钮**: 白色背景，蓝色边框和文本，`12px`圆角
- **危险按钮**: 红色背景，白色文本，`12px`圆角
- **文本按钮**: 无背景，主色文本，无边框
- **图标按钮**: 圆形，`40px`直径，居中图标

#### 输入框

- **标准输入框**: 白色背景，`1px`灰色边框，`8px`圆角，内边距`12px`
- **聚焦状态**: 蓝色边框，轻微阴影
- **错误状态**: 红色边框，错误提示文本
- **禁用状态**: 浅灰色背景，深灰色文本

#### 卡片

- **标准卡片**: 白色背景，`12px`圆角，中等阴影
- **交互卡片**: 悬停时增加阴影深度
- **强调卡片**: 带有顶部彩色边框或背景

#### 图标

- **导航图标**: `24px`大小，线条风格
- **功能图标**: `20px`大小，线条风格
- **指示图标**: `16px`大小，填充风格

### 4.5 交互动效设计原则

- **响应性**: 点击和触摸操作有即时视觉反馈
- **自然过渡**: 使用自然的缓动函数，模拟物理世界的运动
- **适度**: 动效应该增强而非干扰用户体验
- **目的性**: 每个动效都应有明确目的，引导用户注意力

#### 常用动效

- **按钮反馈**: 轻微缩放和颜色变化
- **页面转场**: 淡入淡出或滑动过渡
- **列表动画**: 交错淡入，增强层次感
- **加载动画**: 简洁的旋转或脉冲动画
- **数据更新**: 数值变化时的平滑过渡

## 5. 响应式设计原则

- **移动优先**: 先设计移动端界面，再扩展到大屏幕
- **流动布局**: 使用百分比和弹性布局，适应不同屏幕
- **断点设计**: 在关键断点处调整布局和组件大小
- **内容优先级**: 在小屏幕上优先显示最重要的内容
- **触控友好**: 确保触控目标足够大(至少`44px`)
