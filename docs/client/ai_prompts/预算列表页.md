# 预算列表页

## 页面描述
预算列表页面用于展示和管理用户的所有预算。页面分为两个主要部分：个人预算和通用预算。用户可以在此页面查看预算的基本信息、使用进度和结转情况，并可以添加新的通用预算。

## 页面结构
1. 顶部导航栏：包含返回按钮、页面标题"预算管理"和添加按钮
2. 预算类型选择器：用于切换显示个人预算或通用预算
3. 个人预算列表：显示用户的所有个人月度预算
4. 通用预算列表：显示用户的所有通用预算（长期或无期限）
5. 添加预算按钮：仅在通用预算列表中显示，用于添加新的通用预算

## 数据模型
预算数据模型包含以下关键字段：
- id: 预算ID
- name: 预算名称
- amount: 预算金额
- period: 预算周期（MONTHLY, YEARLY, UNLIMITED）
- budgetType: 预算类型（PERSONAL, GENERAL）
- startDate: 开始日期
- endDate: 结束日期
- rollover: 是否启用结转
- rolloverAmount: 结转金额
- enableCategoryBudget: 是否启用分类预算
- spent: 已使用金额
- remaining: 剩余金额
- progress: 使用进度百分比
- daysRemaining: 剩余天数
- dailySpent: 日均消费
- dailyAvailable: 日均可用

## 交互逻辑
1. 点击预算类型选择器中的按钮，切换显示个人预算或通用预算
2. 点击预算卡片，跳转到对应的预算详情页面
3. 点击预算卡片右上角的更多按钮，显示操作菜单（编辑、删除等）
4. 点击添加按钮，跳转到预算添加页面
5. 点击通用预算列表底部的"添加通用预算"按钮，跳转到预算添加页面

## 业务规则
1. 个人预算每月自动刷新，不能手动添加，只能编辑
2. 通用预算可以手动添加，支持月度、年度或无期限
3. 预算卡片显示预算名称、金额、使用进度、剩余金额和剩余天数
4. 对于启用了结转的预算，显示本月结转金额
5. 对于分类预算，显示对应的分类图标
6. 当预算使用超过80%时，进度条显示为黄色警告
7. 当预算超支时，进度条显示为红色，并显示超支金额

## 页面样式
1. 使用卡片式布局展示预算信息
2. 使用进度条直观显示预算使用情况
3. 使用不同颜色区分正常、警告和超支状态
4. 使用图标表示预算类型和分类
5. 响应式设计，适配不同屏幕尺寸

## API接口
1. GET /api/budgets - 获取预算列表，支持按类型过滤
2. DELETE /api/budgets/:id - 删除指定预算

## 注意事项
1. 确保页面加载时显示加载状态
2. 处理空数据状态，当没有预算时显示提示信息
3. 实现下拉刷新功能，方便用户更新数据
